Preguntas

  • ¿En qué consiste el principio de responsabilidad única? ¿Cuál es su propósito?
    En programación orienta a objetos esto quiere decir que una Clase debe tener un único proposito, por lo que todo el código que se encuentre dentro de esta debe estar orientado a resolver ese único proposito.
  • ¿Qué características tiene según tu opinión “buen” código o código limpio?
    Estas son algunas de las características que en mi opinión debe tener el buen código o código limpio
    • No tener hardcode o codigo "quemado"
    • Una función debe tener un único objetivo e implementar solo este.
    • Una clase debe tener e implementar una única responsabilidad.
    • El código no tiene mas de dos o tres if,while,switch,for,try anidados.
    • Los nombres de las variables deben ser claros y explictos sobre lo que representan.
    • El código debe tener comentarios sobre todo cuando este implementa reglas de negocio que no son simples.
    • Un método o función no superan las 50 lineas de programación.
    • Las clases no superan las 1000 lineas de programación
    • Las lineas de código son cortas, cerca de 10 caracteres como máximo.
